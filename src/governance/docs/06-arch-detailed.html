Key Governance Rules (NTS-GOV-001 v1.1)
GOV-RULE-004
AI explains · never determines
Correct answer is always sourced from DB correct_option. AI explanation engine never overrides or guesses the correct answer.
GOV-RULE-007
verified=1 hard gate
No question is served to any student until an admin explicitly marks it verified=1. Enforced at API level.
GOV-RULE-008
CLAUDE.md updated first
CLAUDE.md must be updated and committed before any new phase or feature work begins. It is the source of truth for scope.
GOV-RULE-009
Explanation engine standards
9 rules enforced via system prompt: no assumed knowledge, no skipped steps, plain language before formula, 3 methods minimum, SVG diagram required on Method 3.
Session Ritual
Start-of-session checklist
Claude Code reads CLAUDE.md → confirms current status → checks admin panel → never uses context when Turso data exists.
Incident Log
Formal incident tracking
3 incidents logged to date: INC-001 (AI used context not DB), INC-002 (admin tasks out of sync), INC-003 (subject section scoped to wrong phase).
